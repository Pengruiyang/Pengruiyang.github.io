jquery原理1

选择函数封装
假定我们提供 jquery 函数, 来完成选择工作
jquery( ‘div > input’ ).each( function () {
this.XXX
})

    可以由两种方案
    1> 封装对象
        function jquery ( selector ) {
            var list = document.querySelectorAll( selector );

            retrun {
                list: list,
                each: function () ...
                map: fucntion () ...
                ... ...
            }
        } 

    2> 封装伪数组
        function jquery( selector ) {
            var arr = { length: 0 };
            // 获取元素
            var list = document.querySelectorAll( selector );

            // 将 list 中的元素加入到 arr 中
            [].push.apply( arr, list );

            // 增加方法
            arr.each = function ( callback ) {
                each( arr, callback );
            };
            return arr;
        }


-> jq 对象本质

  1. 框架核心结构
    -> 核心结构

    1> 带有 new 构造函数的形式
    目标: 
    -> 定义 jquery 构造函数
    -> 利用构造函数创建对象, 即获得页面中的元素( 即, 参数就是选择器 )
    -> 对象应该是一个伪数组, 里面存储着所有的获得到的 DOM 元素
    -> 该对象还需要提供方法, 方法存储到原型中

    优点:
        -> 代码结构紧凑, 不易出错, 书写少很多
        -> 含有隐式迭代, 写起来简单
    缺点:
        -> new 关键字可以考虑消化掉
        -> 隐式迭代太少, 可以考虑可扩展性
        -> 链式编程需要强化

2> 隐藏 new 关键字
    因此只出需要提供工厂函数, 就可以在不写 new 的情况下, 创建 对象了.
    function $( selector ) {
        return new jquery( selector );
    }

    在实际开发中, 常常为了避免全局污染. 会将所有的代码全部放到一个沙箱中.
    因此需要全局暴露, 一个就是 $ 这个函数, 另一个就是 jquery.
    1> 将 $ 暴露是因为开发的需要, 使用 $ 表示与 jquery 中 $ 一致的功能
    2> 将 jquery 暴露出去是因为需要为 jquery 对象 提供 扩展性.

3> 完成框架结构( 初步结构 )
    获得元素使用语法: $( 'div' )
    提供方法使用语法: $.jquery.prototype.XXX = VVV;

    function $( selector ) {
        return new jquery( selector );
    }

    将构造函数作为 $ 这个函数的原型上的一个成员
    function $( selector ) {
        return new jquery( selector );
    }
    $.prototype = {
        jquery: jquery ( selector ) {
            // ...
        }
    }
    目的: 是要可以方便的 给 $.prototype.jquery 这个构造函数 的 prototype 属性添加成员
    将 $.prototype.jquery 的 原型 与 $ 这个函数的原型变成同一个对象

    function $( selector ) {
        return new jquery( selector );
    }
    $.prototype = {
        jquery: jquery ( selector ) {
            // ...
        }
    }
    $.prototype.jquery.prototype = $.prototype;

    如果给 $.prototype 添加成员, 就是在 给 $.prototype.jquery.prototype 添加成员

    实际上给实例对象添加的成员是方法, 所以 jquery 给原型增加了一个名字 fn

    function $( selector ) {
        return new $.fn.jquery( selector );
    }
    $.fn = $.prototype = {
        jquery: jquery ( selector ) {
            // ...
        }
    };
    $.fn.jquery.prototype = $.fn;

    如何要获得元素: $( 'div' )
    如果要增加方法: $.fn.XXX = VVV;

4> 完成框架的核心结构
    -> 如果将 两个变量映射到全局
        在 jq 中可以使用 $, 也可以使用 jQuery 构造函数
    -> 优化代码
        将数组抽取出来, 使用数组方法的时候不要再每次都创建数组对象
        引入 参数 window
        (function ( w ) {
            // ...
            w.document.getElementsByTagName( 'div' )
            w.$ = window.jquery = jquery;
        })( window )

jquery原理2

框架和库是什么?
库: 仓库, 表示的是很多为了实现特殊功能而封装的, 便于使用开发的 函数, 对象 和 常量等.
例如:
// 在 dom 元素下, 获得 复合 tagName 的元素,
// 如果不提供 dom 参数, 那么就是默认 document
function getTag( tagName, dom ) {
dom = dom || document;
return dom.getElementsByTagName( tagName );
}

    getTag( 'div', dv );

    在开发过程中为了实现更加方便的使用, 封装这些方法, 然后组合到一起以供使用, 这样就构成了库.

框架: 是一套完整的开发解决方案. 框架一般提供了一些方便的开发格式与方法, 我们只需要按照他的格式
    使用即可, 不需要关注一些具体的实现.
